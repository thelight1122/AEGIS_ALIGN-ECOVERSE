## Structural Summary
The doctrine family now contains four functional layers:

- Canon layer: Core doctrine and Core candidate
- Origin layer: origin doctrine and origin notes
- Compression layer: seed lines
- Operational layer: recognition heuristics and preamble protocol

Haven's review artifacts strengthen the operational layer and make the doctrine usable beyond AEGIS-native contexts.

## Maintenance Guidance
As the doctrine evolves:

1. Update the Core doctrine first when canonical claims change.
2. Update the seed lines when compressed language stabilizes.
3. Update the heuristics and preamble protocol when operational interpretation changes.
4. Update the origin notes only when new provenance or precursor material is added.
5. Keep this index synchronized whenever a new companion artifact is added or renamed.

## Summary
This index is the threshold map for the relational semiotics doctrine family.

It exists so readers, reviewers, implementers, and future Core custodians can enter the doctrine cluster at the right layer without losing the relationship between canon, origin, compression, and operational use.
