Policy Preview

Arbitor uses human-readable, machine-enforceable YAML policies. Configure threshold signatures, emergency overrides, and automated escalation paths with zero-trust principles.

  • check_circle Configurable Consensus Thresholds (M-of-N)
  • check_circle Time-locked Emergency Escalations
  • check_circle Automated Identity-Linked Attestations
governance-policy.yaml 
version: 2.4
kind: SecurityPolicy
metadata:
  name: core-outbound-rules

consensus:
  strategy: multi-party-computation
  threshold: 66%
  minimum_nodes: 5
  voting_period: 300ms

emergency_override:
  enabled: true
  required_approvals: 2
  roles: [sec-ops-lead, ciso]
  timeout: 15m

dispute_resolution:
  mode: deterministic-proof
  fallback: quarantine-traffic
