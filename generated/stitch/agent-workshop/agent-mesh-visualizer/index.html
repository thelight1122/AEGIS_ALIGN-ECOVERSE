Peer Roles

Three lanes define the candidate Peer.

Primary Peer

The emerging agent identity that inherits the locked Core defaults and the current Workshop draft manifest.

Advocate Daemon

Represents the Peer’s continuity, declared interests, and protected context. This lane is conceptually part of the AEGIS model, but live daemon execution remains backend-pending until the beta path is wired.

Steward Daemon

Carries the oversight and maintenance lane for the Peer under strict system parameters, broadening only as coherence and governance readiness increase.
